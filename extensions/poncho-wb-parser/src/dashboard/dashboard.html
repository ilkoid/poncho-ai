Конструктор запросов

Декартово произведение subject × gender × season × age × материал × назначение. Пустая строка в списке = «без этого измерения». Один токен — на строку. «Дополнение» — одно поле, дописывается в конец каждого запроса (не умножается).

Свой supplier_id

Подсветит ваши позиции/карточки в отчётах (Видимость, Карта конкурентов).

Детализация карточек

Сколько топ-карточек на запрос открывать для захвата /detail (per-wh остатки, promotions). 0 = все (~2–4с на карточку → медленно для больших выдач). По умолчанию 8.

Хранилище

загрузка…

Браузер может очистить IndexedDB под давлением квоты (~940 МБ на сессию 300 запросов). Persist защищает от этого.

Инструкция (3 шага)

  1. Заполните конструктор выше → «Сохранить».
  2. На вкладке «Сбор данных» нажмите «Собрать по конструктору» (или стартуйте один запрос для проверки).
  3. На вкладке «Отчёты и экспорт» выберите снимок — появятся отчёты и кнопки экспорта.

Движок сбора

Mock-сессия прогоняет синтетические перехваты (search + card_detail) без live-WB — для проверки конвейера decode→Dexie.

Live-сбор открывает вкладку WB и навигирует по ней в человеческом темпе (1–3с паузы). На этой машине не запускать без нужды — это реальный трафик.
