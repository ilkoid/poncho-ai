Конструктор запросов

Декартово произведение subject × бренд × gender × season × age × материал × назначение. Пустая строка в списке = «без этого измерения». Один токен — на строку. «Дополнение» — одно поле, дописывается в конец каждого запроса (не умножается).

Фильтр по бренду

Подсветит эти бренды в отчётах (Видимость, Карта конкурентов) — жёлтая строка + ★. Данные не исключает, только акцент.

Детализация карточек

Сколько топ-карточек на запрос открывать для захвата /detail (per-wh остатки, promotions). 0 = все (~2–4с на карточку → медленно для больших выдач). По умолчанию 8.

Браузер может очистить IndexedDB под давлением квоты (~940 МБ на сессию 300 запросов). Persist защищает от этого.

Инструкция (3 шага)

  1. Заполните конструктор выше → «Сохранить». При желании задайте «Фильтр по бренду» для подсветки в отчётах.
  2. На вкладке «Сбор данных» нажмите «Собрать по поиску» (или стартуйте один запрос для проверки).
  3. На вкладке «Отчёты и экспорт» выберите снимок — появятся отчёты и кнопки экспорта.
