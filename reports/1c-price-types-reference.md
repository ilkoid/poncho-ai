## 3. Цены Wildberries (из API)

Цены WB поступают из API Wildberries (Discounts-Prices API). Каждый товар имеет следующие ценовые поля:

| Поле | Описание | Пример |
|------|----------|--------|
| **Розничная цена (price)** | Полная розничная цена без скидок. Аналог ОЭК в 1C. | 1 999 ₽ |
| **Цена со скидкой продавца (discounted_price)** | Цена, которую видит клиент в карточке товара. = Розничная цена − Скидка продавца. **Не включает СПП.** | 1 699 ₽ |
| **Скидка продавца % (discount)** | Процент скидки, которую продавец установил сам. | 15% |
| **Цена WB Club (club_discounted_price)** | Специальная цена для подписчиков WB Club. | 1 599 ₽ |
| **Скидка WB Club % (club_discount)** | Процент скидки для WB Club. | 20% |

### 3.1. Как формируется цена для клиента на WB

```
┌─────────────────────────────────────────────────┐
│  Розничная цена WB (price)                      │  1 999 ₽
│  − Скидка продавца (discount %)                 │  − 15%
│  ─────────────────────────────────────────────── │  ────────
│  = Цена со скидкой продавца (discounted_price)  │  1 699 ₽  ← видна в карточке
│                                                 │
│  При оформлении заказа:                         │
│  − СПП (скидка постоянного покупателя)          │  − ~25%
│  ─────────────────────────────────────────────── │  ────────
│  = Итоговая цена для клиента                    │  ~1 274 ₽  ← видна в корзине
└─────────────────────────────────────────────────┘
```

**Важно:** Скидка продавца и СПП — это **разные скидки**, которые складываются:
- Скидка продавца — видна всем, отображается в карточке товара
- СПП — персональная, применяется только в корзине/при оформлении

### 3.2. СПП из реальных продаж

СПП не приходит напрямую из API цен. Он извлекается из отчёта о продажах:

| Параметр | Описание |
|----------|----------|
| **Источник** | Поле `ppvz_spp_prc` в отчёте о продажах за каждый заказ |
| **Расчёт** | Средний СПП за последние 3 дня по конкретному товару |
| **Покрытие** | Только товары, по которым были продажи за 3 дня |
| **Fallback** | Если продаж нет — используется пользовательский СПП (из параметров отчёта) |

## 4. Сравнение цен 1C и WB: формулы

### 4.1. Какие цены 1C участвуют в сравнении

Из 25 типов цен в сравнении используются **3 типа**:

| Тип цены 1C | Поле в отчёте | Роль |
|-------------|---------------|------|
| Розничная цена ОЭК | `onec_base_price` | Базовая цена для сравнения |
| Розничная цена СР | `onec_sr_price` | Справочно: цена офлайн-канала |
| Спец цена для акции | `is_special_price` | Флаг: товар участвует в акции (0/1) |

Дополнительно вычисляется:
- **ОЭК с СПП 25%** (`onec_spp25_price`) = Розничная цена ОЭК × 0.75

> **Примечание:** В 1C уже существует тип "Розничная цена ОЭК минус СПП 25%" — это то же самое значение, которое вычисляется вручную.

### 4.2. Текущие формулы отклонений

**Отклонение розничной цены (без скидок):**
```
diff_base     = Розничная цена WB − Розничная цена ОЭК
diff_base_pct = diff_base / Розничная цена ОЭК × 100
```

**Отклонение цены со скидкой:**
```
diff_discounted     = Цена WB со скидкой продавца − ОЭК × 0.75
diff_discounted_pct = diff_discounted / (ОЭК × 0.75) × 100
```

### 4.3. Проблемы текущих формул

| # | Проблема | Причина | Влияние на PowerBI |
|---|----------|---------|-------------------|
| **1** | Изменение пользовательского СПП влияет не на тот показатель | ОЭК × 0.75 используется в обоих отклонениях | Пользовательский СПП должен менять "Отклонение СР от ОЭК с СПП", но сейчас меняет оба показателя |
| **2** | Цена ОЭК с реальным СПП не выведена в отчёт | Используется фиксированный СПП 25%, а не реальный из продаж WB | Нет колонки, показывающей ОЭК × (1 − реальный СПП) |
| **3** | Процент отклонения не меняется при изменении параметров | Знаменатель формулы всегда = ОЭК × 0.75 (константа) | При изменении СПП или других параметров % отклонения визуально не меняется |
| **4** | Нет "Цены WB с СПП" | СПП из реальных продаж вычисляется, но не используется для расчёта цены WB с СПП | Невозможно сравнить итоговую цену для клиента с ОЭК |

### 4.4. Требуемые формулы (после исправления)

**Эффективный СПП:**
```
СПП_эффективный = {
    Средний СПП из продаж WB за 3 дня,     если есть продажи
    Пользовательский СПП (из параметров),  если продаж нет
}
```

**Новые расчётные показатели:**
```
ОЭК_с_СПП      = Розничная цена ОЭК × (1 − СПП_эффективный / 100)
WB_цена_с_СПП  = Розничная цена WB × (1 − СПП_эффективный / 100)
```

**Исправленные отклонения:**
```
Отклонение СР от ОЭК с СПП     = Цена WB со скидкой продавца − ОЭК_с_СПП
Отклонение СР от ОЭК с СПП %   = Отклонение / ОЭК_с_СПП × 100
```

## 5. Сводная схема: цены 1C → цены WB

```
┌──────────────────────┐         ┌──────────────────────────────┐
│       ЦЕНЫ 1C        │         │        ЦЕНЫ WILDBERRIES      │
│                      │         │                              │
│ Розничная цена ОЭК   │────────→│ Розничная цена (price)       │
│ (базовая для мартов) │ сравн.  │                              │
│                      │         │ Цена со скидкой продавца     │
│ ОЭК минус СПП 25%   │         │ (discounted_price)           │
│ (уже есть в 1C)      │         │                              │
│                      │         │ Цена WB Club                 │
│ Розничная цена СР    │ справ.  │ (club_discounted_price)      │
│ (офлайн магазины)    │         │                              │
│                      │         │ СПП из продаж                │
│ Спец цена для акции  │ флаг    │ (ppvz_spp_prc → avg за 3 дня)│
│ (участие в акции)    │         │                              │
└──────────────────────┘         └──────────────────────────────┘
        │                                    │
        └────────────┬───────────────────────┘
                     ▼
         ┌───────────────────────┐
         │   ОТЧЁТ POWERBI       │
         │   (price_comparison)  │
         │                       │
         │  • diff_base          │
         │  • diff_discounted    │
         │  • diff_discounted_pct│
         │  • статус (match/     │
         │    warning/overprice) │
         └───────────────────────┘
```

1. СПП из продаж WB за 3 дня — приоритет 1
2. Пользовательский СПП — приоритет 2 (если продаж нет)

Ассортиментный средний СПП исключается из расчётов?

### В2. Цена WB со скидкой vs Цена WB с СПП

Цена со скидкой продавца (`discounted_price`) — это цена **без СПП** (видна в карточке товара).

СПП — это **дополнительная** скидка, которая применяется в корзине.

**Вопрос:** Верно ли, что:
- Цена со скидкой продавца = цена для клиента **без СПП**
- Цена со скидкой продавца × (1 − СПП/100) = цена для клиента **с СПП** (итоговая)

Именно итоговая цена (с СПП) должна сравниваться с ОЭК?

### В3. "Отклонение СР от РЦ WB" — определение

**Вопрос:** "Отклонение СР от РЦ WB" в отчёте PowerBI — это:
- **Вариант А:** Цена со скидкой продавца − Розничная цена WB (размер скидки WB в рублях, всегда отрицательное)
- **Вариант Б:** Розничная цена WB − Розничная цена ОЭК (разница розничных цен WB vs 1C)
- **Вариант В:** что-то другое?
