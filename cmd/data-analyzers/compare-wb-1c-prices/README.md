# compare-wb-1c-prices

Сравнение цен 1C и Wildberries для финансового директора (PowerBI отчёт).

Читает данные из `wb-sales.db`, объединяет 1C и WB по цепочке `onec_goods.article → cards.vendor_code`, вычисляет отклонения цен и записывает результат в BI-базу.

## Запуск

```bash
go run .                          # последние снимки из каждой системы
go run . --date 2026-04-08        # конкретная дата
go run . --force                  # пересчитать даже если результаты есть
go run . --csv                    # экспорт в CSV
go run . --output /tmp/bi.db      # указать путь к BI-базе
```

## Источники данных

| Поле | Источник | Описание |
|------|----------|----------|
| Цена ОЭК | 1C `onec_prices` тип «Розничная цена ОЭК» | Базовая розничная цена |
| Цена СР | 1C `onec_prices` тип «Розничная цена СР» | Цена офлайн-магазинов |
| Спец. цена | 1C `onec_prices` тип «Спец цена для акции» | Флаг: 1 = акция активна |
| Цена WB | WB `product_prices.price` | Розничная цена WB |
| Цена WB со скидкой | WB `product_prices.discounted_price` | Цена после скидки продавца |
| Скидка продавца | WB `product_prices.discount` | Скидка продавца % |
| СПП за 3 дня | `AVG(sales.ppvz_spp_prc)` за 3 дня | Фактический СПП из продаж |
| Остатки WB | `products.stock_wb + stock_mp` | Общие остатки |

## Формулы расчёта

### Цепочка цен WB

```
price → ×(1-discount%) → discounted_price → ×(1-СПП%) → цена с СПП → ×(1-кошелёк%) → цена с кошельком
```

### Вычисляемые поля

| Поле | Формула |
|------|---------|
| ОЭК с СПП | `ОЭК × (1 − СПП_пользователя)` — всегда пользовательский СПП (14-40%) |
| РЦ WB с СПП | `Цена_со_скидкой × (1 − СПП)` — СПП из API при наличии, иначе пользовательский |
| РЦ WB с кошельком | `РЦ_WB_СПП × (1 − кошелёк%)` |
| Цена СР с лояльностью | СР (если спец.цена=1) или `СР × (1 − лояльность%)` (если спец.цена=0) |

### Отклонения

| Метрика | Формула |
|---------|---------|
| Отклонение WB от ОЭК | `Цена_WB_со_скидкой − ОЭК` |
| Отклонение WB от ОЭК % | `(Отклонение / ОЭК) × 100` |
| Отклонение СР от РЦ WB | `СР_лояльность − РЦ_WB_кошелёк` |
| Отклонение СР от РЦ WB % | `(Отклонение / СР_лояльность) × 100` |
| Отклонение СР от ОЭК с СПП | `СР_лояльность − ОЭК_с_СПП` |
| Отклонение СР от ОЭК с СПП % | `(Отклонение / СР_лояльность) × 100` |

## Параметры конфигурации

```yaml
comparison:
  spp_user_percent: 25              # Пользовательский СПП (14-40%), для ОЭК
  wb_wallet_percent: 3              # WB кошелёк %: 2, 3 или 5
  loyalty_discount_percent: 10      # Скидка «Яркие вместе»: 5.5, 10, 13.5, 19, 25, 50
  spp_days_back: 3                  # Окно расчёта СПП из продаж (дни)
  match_threshold: 50               # |отклонение| ≤ 50₽ → "match"
  warning_threshold: 500            # |отклонение| ≤ 500₽ → "warning"
```

## Выходные данные

Таблица `price_comparison` в BI-базе (`bi.db`), 64 колонки:
- Идентификация, товарная иерархия, атрибуты
- Сырые цены 1C и WB
- Расчётные цены (с СПП, с кошельком, с лояльностью)
- Отклонения (абсолютные и %)
- Статусы: `match`, `warning`, `overpriced`, `underpriced`

## Не реализовано

- Остатки из 1С по подразделениям (Внуково, ОПИП, Франчайзи, ОРП)
- Остатки из ЛК по каналу ГиперГлобус

Эти данные отсутствуют в текущей БД и требуют новых загрузчиков.
