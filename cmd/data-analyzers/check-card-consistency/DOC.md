# Проверка качества карточек товаров

## Назначение

Утилита автоматически проверяет карточки товаров на Wildberries на наличие расхождений между тем, что показано на фотографиях, и тем, что заполнено в характеристиках. Незаполненные, некорректные или противоречивые характеристики снижают видимость товаров в поиске и ухудшают конверсию. Утилита находит такие проблемы и формирует готовые исправления.

## Как это работает

Обработка идёт поэтапно — каждый этап можно запускать отдельно, результаты предыдущих сохраняются. Повторный запуск пропускает уже обработанные карточки.

**Этап 0 — Предпросмотр.** Показывает, сколько карточек попадёт в обработку после фильтрации, и какие у них проблемы. Позволяет оценить объём и стоимость анализа до его запуска.

**Этап 1 — Аудит.** Основной этап. Визуальная модель анализирует фотографии товара вместе с заполненными характеристиками и находит расхождения — несоответствия цвета, состава, типа ткани, комплектности, назначения и других свойств.

**Этап 2 — Метрики.** Подтягивает дополнительные данные по карточкам: рейтинг, количество отзывов, видимость в поиске, поисковые запросы. Не требует обращения к платным API и работает быстро.

**Этап 4 — Генерация исправлений.** Для карточек с найденными расхождениями формируются корректные значения характеристик на основе данных аудита и справочника характеристик Wildberries.

**Этап 5 — Обновление карточек.** Применяет сгенерированные исправления к карточкам товаров через API Wildberries. Сейчас работает в тестовой среде — токен для контентного API в периоде охлаждения после смены администратора аккаунта.

## Фильтрация

Перед обработкой можно сузить список карточек. Фильтры комбинируются:

| Фильтр | Что делает |
|--------|-----------|
| По артикулу товара | Выбрать конкретные товары по их WB-артикулам |
| По артикулу продавца | Выбрать товары по внутренним SKU-кодам |
| По году коллекции | Ограничить товары определённым годом (извлекается из артикула продавца) |
| По сезону | Выбрать товары определённого сезона: весна, лето, осень, зима |
| По категории | Ограничить конкретной товарной категорией Wildberries |
| Только в наличии | Оставить только товары с остатками на складах |
| По проблеме | Выбрать карточки с расхождениями, ошибками обработки или ожидающие обновления |
| По рейтингу/видимости | Оставить карточки с низким рейтингом, малым числом отзывов или низкой видимостью |

Фильтр по проблемам позволяет точечно работать с карточками: доаудитить те, где были ошибки, или обновить те, где исправления уже сгенерированы, но ещё не применены.

## Аудит карточек

Визуальная модель получает до 5 фотографий товара, его название, описание и заполненные характеристики. На основе визуального анализа определяются:

- **Тип изделия** — соответствует ли заявленный тип тому, что на фото
- **Цвет** — доминирующий цвет указан первым, дополнительные цвета корректны
- **Свойства ткани** — например, «эластичный» не должно быть у тканей, которые не тянутся
- **Комплектность** — указан набор или отдельное изделие в соответствии с фото
- **Целевая аудитория** — определяется по модели и размерному ряду
- **Пустые характеристики** — если свойство можно определить по фото, оно должно быть заполнено

По результатам аудита карточка получает заключение: есть расхождения или нет, и если есть — подробный список проблем с указанием текущего и корректного значения.

## Генерация исправлений

Для карточек с расхождениями утилита запрашивает справочник характеристик Wildberries для подходящей категории товаров. Модель получает:

- Результаты аудита со списком проблем
- Справочник допустимых характеристик и их значений для данной категории
- Текущие характеристики карточки

На основе этих данных формируются корректные значения — строго в формате, который принимает Wildberries. Системные поля (штрихкоды, коды ТН ВЭД, сертификаты) сохраняются без изменений.

## Обновление карточек

Сгенерированные исправления применяются через API Wildberries. Реализовано два режима:

- **Тестовый** — изменения применяются в песочнице Wildberries, где можно проверить корректность без влияния на реальные карточки. Это текущий рабочий режим, так как основной токен находится в периоде охлаждения после смены администратора аккаунта.
- **Рабочий** — обновление реальных карточек. Требует дополнительного подтверждения. Применяется только после проверки в тестовой среде.

Все изменения логируются — можно отследить, что именно было изменено в каждой карточке, и при необходимости откатить.

## Экспорт результатов

Результаты анализа выгружаются в Excel-файл со следующей информацией:

- Фотография товара и ссылка на карточку на Wildberries
- Название, артикулы, категория, сезон
- Метрики качества: рейтинг, отзывы, видимость в поиске
- Поисковые запросы, по которым товар находят покупатели
- Найденные расхождения и их описание
- Сгенерированные исправления: новое название, описание, характеристики
- Статус обработки по каждому этапу

Ячейки с расхождениями и низкими показателями выделяются цветом для быстрого визуального анализа.
