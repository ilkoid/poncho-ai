## 1. Идеальный шторм: товар кончается, спрос растёт, поставок нет

> Таблица: `risk_ideal_storm` — 267 строк

**Что показывает:** Размеры, которые закончатся через 1–7 дней, при этом спрос растёт (trend_pct > 0), а incoming-поставок нет. Это приоритет №1 — если сегодня не создать поставку, завтра будет упущенная выручка.

```sql
SELECT
  s.name || ' (' || s.tech_size || ')' AS товар_размер,
  s.brand,
  s.category,
  s.region_name AS федеральный_округ,
  s.stock_qty AS остаток_шт,
  ROUND(s.ma_3, 2)  AS продажи_3дня,
  ROUND(s.ma_7, 2)  AS продажи_7дней,
  ROUND(s.trend_pct, 0) AS тренд_процент,
  ROUND(s.sdr_days, 2) AS дней_до_обнуления,
  COALESCE(s.supply_incoming, 0) AS в_пути_шт
FROM ma_sku_daily s
WHERE s.critical = 1
  AND (s.supply_incoming IS NULL OR s.supply_incoming = 0)
  AND (s.trend_pct IS NULL OR s.trend_pct > 0)
ORDER BY s.ma_7 DESC;
```

**Пример:** Джинсы PlayToday Kids р.122 — 5 шт, продаётся 4 шт/день, тренд +17%, поставок 0 → кончатся через 1.8 дня.

## 2. Упущенная выручка: товар закончился, а его покупают

> Таблица: `risk_lost_revenue` — 2 135 строк

**Что показывает:** Размеры, которых уже нет на складе, но по которым есть стабильный спрос (ma_7 > 0). Колонка `потеря_за_2недели_шт` — сколько единиц вы не продадите за 2 недели, если не привезёте товар.

```sql
SELECT
  s.name || ' (' || s.tech_size || ')' AS товар_размер,
  s.brand,
  s.category,
  s.region_name AS федеральный_округ,
  ROUND(s.ma_7, 2) AS спрос_шт_в_день,
  ROUND(s.ma_7 * 14, 2) AS потеря_за_2недели_шт,
  COALESCE(s.supply_incoming, 0) AS в_пути_шт,
  s.season
FROM ma_sku_daily s
WHERE s.out_of_stock = 1
  AND s.ma_7 > 0
ORDER BY s.ma_7 DESC;
```

**Пример:** Джинсы PlayToday Kids р.110 — 0 на складе, спрос 2.29 шт/день → ~32 единицы потеряно за 2 недели.

## 3. Выбитая размерная сетка у продающих товаров

> Таблица: `risk_broken_grid` — 5 826 строк

**Что показывает:** Карточки, у которых часть размеров выбита, при этом карточка активно продаётся. Покупатель видит «размера нет» — уходит к конкуренту. Нужно восполнить именно эти размеры, а не карточку целиком.

```sql
SELECT
  s.name,
  s.brand,
  s.category,
  s.region_name AS федеральный_округ,
  s.total_sizes AS всего_размеров,
  s.sizes_in_stock AS в_наличии,
  ROUND(s.fill_pct, 2) AS заполненность_процент,
  ROUND(s.ma_7, 2) AS продажи_шт_в_день,
  s.stock_qty AS остаток_шт
FROM ma_sku_daily s
WHERE s.broken_grid = 1
  AND s.ma_7 > 0
ORDER BY s.ma_7 DESC;
```

**Пример:** Джинсы PlayToday Kids — 3 из 5 размеров (60%), продаётся 5.29 шт/день. Каждый выбитый размер ≈ потеря 20% конверсии.

## 4. Всплеск спроса: продажи резко выросли

> Таблица: `risk_demand_surge` — 6 279 строк

**Что показывает:** Размеры, у которых продажи за последние 3 дня значительно превысили среднюю за 7 дней (trend_pct > 50%). Если SDR маленький — это следующий кандидат в «критичные» через 2–3 дня.

```sql
SELECT
  s.name || ' (' || s.tech_size || ')' AS товар_размер,
  s.brand,
  s.category,
  s.region_name AS федеральный_округ,
  ROUND(s.ma_3, 2) AS продажи_3дня,
  ROUND(s.ma_7, 2) AS продажи_7дней,
  ROUND(s.trend_pct, 2) AS тренд_процент,
  s.stock_qty AS остаток_шт,
  ROUND(s.sdr_days, 2) AS дней_до_обнуления,
  COALESCE(s.supply_incoming, 0) AS в_пути_шт
FROM ma_sku_daily s
WHERE s.trend_pct IS NOT NULL
  AND s.trend_pct > 50
  AND s.stock_qty > 0
ORDER BY s.trend_pct DESC;
```

**Пример:** Футболка PlayToday Tween р.164 — тренд +133%, 36 шт, SDR=50 дней. Пока запас есть, но стоит взять на контроль.

## 5. Сводка рисков по бренду и категории

> Таблица: `risk_brand_category_summary` — 206 строк

**Что показывает:** «Тепловая карта» проблемных направлений. Какой бренд × категория «горит» сильнее всего — по количеству критичных, рискованных и отсутствующих позиций.

```sql
SELECT
  brand AS бренд,
  category AS категория,
  COUNT(*) AS всего_ску,
  SUM(CASE WHEN critical = 1 THEN 1 ELSE 0 END) AS критичные,
  SUM(CASE WHEN risk = 1 THEN 1 ELSE 0 END) AS рискованные,
  SUM(CASE WHEN out_of_stock = 1 AND ma_7 > 0 THEN 1 ELSE 0 END) AS отсутствует_со_спросом,
  SUM(CASE WHEN broken_grid = 1 AND ma_7 > 0 THEN 1 ELSE 0 END) AS выбитая_сетка,
  ROUND(AVG(fill_pct), 2) AS средняя_заполненность_процент
FROM ma_sku_daily
GROUP BY brand, category
HAVING критичные + рискованные + отсутствует_со_спросом > 0
ORDER BY (критичные + рискованные + отсутствует_со_спросом) DESC;
```

**Пример:** Джинсы*М Tween — 26 критичных, 87 отсутствующих со спросом, заполненность 56%. Полуботинки Tween — 106 отсутствующих. Это два самых проблемных направления.

## 6. Неликвид: много на складе, никто не покупает

> Таблица: `risk_dead_stock` — 2 640 строк

**Что показывает:** Размеры с остатком ≥30 шт, у которых ma_28 ≈ 0 — спроса нет уже месяц. Эти единицы замораживают оборотный капитал и занимают склад. Решение: скидка, перемещение, вывод из ассортимента.

```sql
SELECT
  s.name || ' (' || s.tech_size || ')' AS товар_размер,
  s.brand,
  s.category,
  s.season,
  s.region_name AS федеральный_округ,
  s.stock_qty AS остаток_шт,
  ROUND(s.ma_28, 2) AS средние_продажи_28дней,
  ROUND(CASE WHEN s.ma_28 > 0 THEN s.stock_qty / s.ma_28 ELSE NULL END, 2) AS запас_в_днях
FROM ma_sku_daily s
WHERE s.stock_qty >= 30
  AND (s.ma_28 IS NULL OR s.ma_28 < 0.1)
ORDER BY s.stock_qty DESC;
```

**Пример:** Кепка PlayToday Tween — 218 шт, ma28=0.04 → запас на 6100 дней. Сумка меховая Adults — 210 шт, нулевой спрос. Это «Осень», товар не по сезону.

## 7. Сезонный риск: склад забит не-весенними товарами

> Таблица: `risk_season_mismatch` — 90 строк

**Что показывает:** Сейчас апрель (весна), но на складах лежат десятки тысяч единиц «Осень» и «Школа» с околонулевым спросом. Нужно разделить: норма (закупка под новый сезон) от неликвида (неверно спрогнозированный ассортимент).

```sql
SELECT
  s.season AS сезон,
  s.category AS категория,
  COUNT(*) AS количество_ску,
  SUM(s.stock_qty) AS всего_на_складе,
  ROUND(AVG(s.ma_7), 2) AS средние_продажи_шт_в_день,
  ROUND(AVG(s.fill_pct), 2) AS средняя_заполненность_процент,
  SUM(CASE WHEN s.ma_7 IS NULL OR s.ma_7 = 0 THEN 1 ELSE 0 END) AS ску_без_спроса
FROM ma_sku_daily s
WHERE s.season != 'Весна'
GROUP BY s.season, s.category
HAVING SUM(s.stock_qty) > 50
ORDER BY SUM(s.stock_qty) DESC;
```

**Пример:** Осень:Термокомплекты — 12 514 шт на складе, из 1 138 SKU лишь 49 продаются. Школа:Брюки — 6 779 шт, 614 из 659 SKU без продаж.

## 8. Региональный дисбаланс: дефицит в одном ФО, избыток в другом

> Таблица: `risk_regional_imbalance` — 1 113 строк

**Что показывает:** Один и тот же размер — дефицит на Урале (1–2 шт) и избыток в Центральном (20–40 шт без движения). Межрегиональное перемещение решает проблему без новой поставки от производителя.

```sql
SELECT
  s.name,
  s.brand,
  s.tech_size AS размер,
  MAX(CASE WHEN s.stock_qty <= 2 AND s.ma_7 > 0 THEN s.region_name END) AS регион_дефицит,
  MAX(CASE WHEN s.stock_qty <= 2 AND s.ma_7 > 0 THEN s.stock_qty END) AS остаток_дефицит,
  MAX(CASE WHEN s.stock_qty > 20 AND (s.ma_7 IS NULL OR s.ma_7 < 0.5) THEN s.region_name END) AS регион_избыток,
  MAX(CASE WHEN s.stock_qty > 20 AND (s.ma_7 IS NULL OR s.ma_7 < 0.5) THEN s.stock_qty END) AS остаток_избыток
FROM ma_sku_daily s
GROUP BY s.nm_id, s.chrt_id
HAVING регион_дефицит IS NOT NULL AND регион_избыток IS NOT NULL
ORDER BY остаток_дефицит ASC;
```

**Пример:** Ботинки Tween р.38 — дефицит на Урале (1 шт), избыток в Центральном (30 шт). Толстовка р.146 — Урал (1 шт) vs Центральный (45 шт).
