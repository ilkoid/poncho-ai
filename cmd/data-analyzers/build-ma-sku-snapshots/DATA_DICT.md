# SKU Snapshots — Справка по полям (ma_sku_daily)

Одна строка = один размер (chrt_id) одного товара в одном федеральном округе на конкретную дату.

## Идентификаторы

- **snapshot_date** — дата снапшота (обычно вчера)
- **nm_id** — артикул WB (номенклатура)
- **chrt_id** — конкретный размер внутри nm_id
- **region_name** — федеральный округ (Центральный, Приволжский и т.д.)
- **article / vendor_code** — артикул продавца из 1C/PIM
- **identifier** — внутренний код товара

## Атрибуты товара

Справочные поля из карточек WB и 1C: name, brand, type, category, sex, season, color, collection. Источник — сырые данные, без расчёта.

## Остатки

- **stock_qty** — штук на складах в этом ФО (quantity + возвраты от клиентов). Источник: stocks_daily_warehouses
- **supply_incoming** — штук в пути (из поставок: заказано − уже на складе). Источник: supplies
- **total_sizes** — всего размеров у карточки. Источник: cards_sizes
- **sizes_in_stock** — сколько размеров есть в наличии в этом ФО (с дедупликацией по складам). Расчёт: count(distinct chrt_id where stock > 0)
- **fill_pct** — заполненность размерного ряда, %. Расчёт: sizes_in_stock / total_sizes × 100

## Скользящие средние (MA)

Среднедневные продажи за период, только по этому размеру в этом ФО. Источник: таблица sales, расчёт MA за окно.

- **ma_3** — за 3 дня (короткий всплеск)
- **ma_7** — за 7 дней (неделя, базовый показатель)
- **ma_14** — за 14 дня
- **ma_28** — за 28 дней (месяц)

Если продаж в регионе нет — поле пустое (NULL).

## Производные метрики

- **sdr_days** — на сколько дней хватит остатка. Расчёт: stock_qty / ma_7. Чем меньше — тем быстрее кончится
- **trend_pct** — тренд спроса, %. Расчёт: (ma_3 − ma_7) / ma_7 × 100. Положительный = спрос растёт, отрицательный = падает

## Флаги (0/1)

- **critical** — критично мало: остаток есть, но SDR ≤ 7 дней
- **risk** — риск дефицита: остаток есть, но SDR ≤ 12 дней
- **out_of_stock** — нет в наличии: stock_qty ≤ порога (по умолчанию = 0) и есть спрос (MA-7 > 0)
- **broken_grid** — выбит размерный ряд: sizes_in_stock < total_sizes
