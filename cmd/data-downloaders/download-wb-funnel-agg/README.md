# download-wb-funnel-agg

Утилита для загрузки агрегированных данных воронки продаж из WB Analytics API v3.

## Назначение

Загружает агрегированные метрики воронки за период с возможностью сравнения с прошлым периодом.

**Отличие от обычного funnel:**
- `funnel` — ежедневные данные для графиков (временные ряды)
- `funnel-agg` — агрегация за период для сравнений и рейтингов

## Данные

Загружает ~60+ метрик:
- Базовые: просмотры, корзины, заказы, выкупы, избранное
- Дополнительные: отмены, средний чек, время сборки, WB Club
- Сравнение: динамика между selected и past периодами

## Использование

### Еженедельная загрузка (рекомендуется)

Каждый понедельник загружаем данные за прошлую неделю (Пн-Вс) и сравниваем с позапрошлой неделей.

**Шаг 1. Обновить даты в config.yaml:**
```bash
./update-funnel-agg-dates.sh
```

Скрипт автоматически вычисляет:
- Selected: прошлая неделя (Пн-Вс)
- Past: позапрошлая неделя (Пн-Вс)

**Шаг 2. Запустить загрузку:**
```bash
cd cmd/data-downloaders/download-wb-funnel-agg && go run main.go
```

### Конфигурация

Файл: `config.yaml`

```yaml
funnel_aggregated:
  # Даты обновляются скриптом update-funnel-agg-dates.sh
  selected_start: "2026-03-23"  # Понедельник прошлой недели
  selected_end:   "2026-03-29"  # Воскресенье прошлой недели
  past_start: "2026-03-16"  # Понедельник позапрошлой недели
  past_end:   "2026-03-22"  # Воскресенье позапрошлой недели

  page_size: 1000  # Товаров за запрос (max 1000)
  db_path: "db/wb-sales.db"
```

### Переменные окружения

```bash
export WB_API_ANALYTICS_AND_PROMO_KEY=your_key_here
```

## Хранение данных

Таблица: `funnel_metrics_aggregated`

UNIQUE constraint: `(nm_id, period_start, period_end)`

**Важно:** Каждый запуск с новыми датами создаёт новые записи. Данные накапливаются по неделям.

## Скрипт обновления дат

`update-funnel-agg-dates.sh` — обновляет даты в config.yaml для прошлой недели.

Вычисляет правильные даты Пн-Вс независимо от дня запуска.

## Rate limiting

Analytics API v3: 3 req/min (очень медленно!)

Утилита использует адаптивный rate limiting — автоматически выдерживает задержки между запросами.

## Отличия от funnel (history)

| Параметр | funnel | funnel-agg |
|----------|--------|------------|
| API endpoint | `/products/history` | `/products` |
| Гранулярность | По дням | За период |
| Строк за 7 дней | ~686,000 | ~19,532 |
| Для чего | Графики трендов | Сравнение периодов |
